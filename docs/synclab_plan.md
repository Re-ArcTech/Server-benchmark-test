# 同期ラボ（SyncLab）実装計画書

位置同期の各手法を**トグルで切り替えて体感・比較するデバッグツール**。
技術検証用（本番ではない）。ベンチリポジトリ(Server-benchmark-test)に同居。

## 目的
- 「権威モデル × 見せ方」の組み合わせを、遅延を注入しながら実際に動かして比較する
- どの手法がどれだけ効くかを**赤(生データ)/青(処理後)の見える化**で確認する

## 軸（前回整理ずみ）
- **権威**: クライアント / ハイブリッド(予測+訂正) / サーバー
- **他人の見せ方**: 直接 / 補間(バッファ+ディレイ)
- **自分の見せ方**: 直接 / 予測
- **訂正**: なし / あり（予測時のみ）
- **遅延注入**: 0〜300ms（手法の差を出すため）

## サーバー(Go) 仕様

`GET /synclab`（WebSocket）。port 8090。

クライアント→サーバー:
- `config` `{authority, sendRateHz, latencyMs}` … デバッグUIから動的変更
- `move` `{seq, pos, rotY, vel}` … クライアント/ハイブリッド権威時の予測位置
- `input` `{seq, dir, dt}` … サーバー権威時の入力（サーバーが動かす）

サーバー→クライアント（すべて `latencyMs` 遅延を乗せて送信）:
- `bot.state` `{pos, rotY, vel}` … BOTが8の字に動く（=観測用の他者）。`sendRateHz`で送信
- `self.echo` `{seq, pos}` … クライアント権威: 受け取った位置をそのまま返す（自分の遅延を見る用）
- `self.auth` `{seq, pos, vel}` … サーバー権威: サーバーが入力から計算した位置
- `self.correction` `{ackSeq, pos, vel}` … ハイブリッド: ズレが閾値超えたら訂正

権威モード別のサーバー挙動:
- **クライアント権威**: `move`を受けてそのまま`self.echo`で返す（中継のみ）
- **サーバー権威**: `input`を受けて、サーバー側で等速移動を計算→`self.auth`を返す
- **ハイブリッド**: `move`(予測位置)を受け、フィールド外などズレを検知したら`self.correction`を返す

## Unity 仕様

### シーン
- 床(Plane) + 自キャラ(青Capsule) + BOT(緑Capsule)
- ゴースト表示: 生の受信位置を半透明の赤Capsuleで重ねて表示（処理前後を並べて見る）

### 操作
- 自キャラ: WASD/矢印で移動（軽い等速移動。物理は使わない＝検証をシンプルに）

### デバッグパネル(IMGUI)
- 権威: クライアント/ハイブリッド/サーバー（ラジオ）
- 他人表示: 直接/補間 + 補間ディレイ スライダー(0-300ms)
- 自分表示: 直接/予測
- 訂正: ON/OFF
- 遅延注入: スライダー(0-300ms) → サーバーへconfig送信
- 送信レート: スライダー(10-60Hz)
- **プリセット4ボタン**: ①素朴 ②クライアント+補間 ③ハイブリッド ④サーバー権威
- 統計: RTT / 訂正回数 / バッファ段数

### 見える化
- BOT: 赤ゴースト=最後に受信した生位置 / 緑=表示位置(補間後)
- 自分: 赤ゴースト=サーバーecho/auth位置 / 青=予測表示位置
- → 赤と表示の「ズレ」で各手法の効果が一目で分かる

## スコープ（v1）
- **対象はプレイヤー位置のみ**（ボールは入れない＝検証をシンプルに）
- 2人目は**サーバーBOT**（毎回同じ動きで手法を公平比較）。ParrelSyncでの実2人対戦は後で
- 訂正は**まず「スナップ訂正」**（ズレたら即補正）。フルなロールバック再計算は後で

## 段階
```
1. Goリレーサーバー（BOT・config・3権威モード）→ 単体で疎通確認
2. Unity: 接続・送受信・自キャラ移動
3. Unity: BOT表示（直接→補間→ゴースト見える化）
4. Unity: 自キャラ予測→訂正
5. デバッグパネル＋4プリセット
6. 遅延注入しながら4プリセット比較
```
